Sounds Of Serenity.

The Idea...

After creating the first 'Sounds of Serenity' website, it worked as intended, but it was not coded with scalability in mind, nor did it look how I envisioned it. I took it upon myself to completely redesign the app, and embarked on transitioning it to a full-stack application. The end goal of the website is for users to be able to log in safely , create playlists of their favorite tracks and listen to music to help with the meditation practice.

Main Page

As you can see from the comparison on the left, the new site is much more colorful and has a different user interface. When more tracks are added on the server, the 'recently added' and 'collection' sections are automatically updated with the four newest to not overwhelm the user when they log in. As I develop the app to contain more music, I will add a new page which will contain all the collections which will be searched for by the user.

Only users with a valid accout can acesss this page. A user without an account will be redirected to the login Page.

Login Page

A user can make an account and log in safely, with their information is stored on a Mongo database. Passwords on the mongoDB are encrypted to provide a high level of secruity. A user is given a JWT token upion login and can acess the main page.

Player Page

When a user clicks on a track, they are presented with a player page with simple music controls. The background changes dependent on the track image seen previously on the main page and it is easy to navigate back to the main page via the upward arrow.

Collection Page

The collection page allows users to pick a certain 'mood' to listen to. When a collection is chosen they are given the option to listen any track within that specific collection and are directed to the player page upon choosing. The User Experience flow is very simple, main page => collections => player

Mobile Responsive

I can envisage most users using the app on the mobile devices so it was extremely important to make sure the app was initially designed on mobile devices and translated to desktop.
